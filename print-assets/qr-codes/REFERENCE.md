# NBD Print-Tracking QR Codes — Reference

Generated by [`scripts/generate_qrs.py`](../../scripts/generate_qrs.py).
Regenerate any time the script's `PIECES` table changes:

```bash
pip install "qrcode[pil]"
python scripts/generate_qrs.py
```

All 7 QRs are 1200×1200 PNG, pure black on pure white, error
correction **H** (~30% — survives dirt, scuffs, and sun fade).
Modules are rendered at 18 px each (no resize — every module is a
crisp integer pixel block), then centered in the 1200 canvas with
white padding that doubles as extra quiet zone.

## Printed-size guidance

Always print on a **pure white background** with a **"Scan Me"** label
nearby (a QR with no human-readable affordance gets ignored).

| Piece | Minimum printed QR size |
|---|---|
| Business cards, stickers | **0.8"** (20 mm) |
| Door hangers             | **1.5"** (38 mm) |
| Yard signs, banners      | **6"** (150 mm) or larger |

These are floors, not targets — bigger is always better. The error
correction is set to H specifically so banners that get rained on or
business cards that get pocketed still scan.

## Updating destinations later

- **/inspect URLs:** the route is permanent. UTM params live in the
  QR — to change attribution, regenerate the QR with new UTMs.
- **/r destination:** edit the `destination` field in `firebase.json`
  for the `/r` redirect and redeploy. The QRs themselves don't need
  to change. (302 not 301 was chosen here on purpose so old browser
  caches don't pin to an outdated target.)
