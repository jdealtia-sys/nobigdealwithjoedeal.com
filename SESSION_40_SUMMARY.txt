✅ WHAT WAS FIXED:

1. **Auth State Settling**
   - loadLeads() now waits 1 second if window._user is undefined
   - Retries once before failing
   - Prevents race condition where Firestore query runs before auth completes

2. **Enhanced Diagnostics**
   - Console logging shows: uid, auth state, Firestore query status, doc count
   - Exposes exact failure point: auth, query, or rules
   - Full error object logged on failures

3. **Visual Loading State**
   - Kanban shows "⏳ Loading leads..." while query runs
   - Prevents user from thinking page is frozen
   - Replaced with actual kanban or diagnostic panel when complete

4. **Diagnostic Panel (Orange Warning Box)**
   - Auto-appears when: zero leads + user authenticated
   - Shows: user email, uid, database status, possible causes
   - Quick actions: Load Sample Data, Reload Page, Dismiss
   - Guides user to troubleshoot Firestore rules or network issues

5. **Load Sample Data Button (Purple)**
   - Auto-shows when zero leads exist
   - Creates 5 realistic test leads:
     * Different stages (New → In Progress)
     * Cincinnati-area addresses
     * Multiple carriers (State Farm, Allstate, Liberty Mutual, etc.)
     * Different damage types (Hail, Wind, Siding, Full Exterior, Gutters)
     * Realistic job values ($3,200 - $24,500)
   - Auto-hides after leads are created
   - One-click fix for empty account testing

6. **Firestore Rules Template**
   - Complete security rules for all NBD Pro collections
   - User-scoped access (userId == auth.uid)
   - Deployment instructions in FIRESTORE_RULES.txt
   - Without these rules, CRM is completely broken
