WHAT WAS BUILT
================================================================================

A complete multi-tenant company isolation system for the NBD Pro CRM that enables
multiple roofing companies to use the same platform with:

  ✓ Individual company branding (colors, logo, name)
  ✓ Automatic company detection from user profiles
  ✓ Company-aware data storage and retrieval
  ✓ Intelligent notification routing to company owners
  ✓ Seamless user experience with zero additional login steps

FIRESTORE SCHEMA
================================================================================

Two Firestore collections with company isolation:

companies/{companyId}
  - id: string (unique ID)
  - name: string (display name)
  - owner: string (contact person)
  - phone: string (notification phone)
  - email: string (notification email)
  - address: string (physical address)
  - logo: string|null (Cloud Storage URL)
  - colors: object (primary, accent, navBg hex values)
  - services: array (list of services offered)
  - serviceAreas: array (geographic service areas)
  - warranty: string (warranty text)
  - subscription: object (plan type, status)
  - createdAt: Timestamp
  - updatedAt: Timestamp
  - siteUrl: string (public website URL)

users/{uid}
  [UPDATED TO ADD]
  - companyId: string (links to company)
  [All other fields unchanged]

API QUICK REFERENCE
================================================================================

CompanyAdmin Module (pro/js/company-admin.js)
  
  CompanyAdmin.getCompanyConfig(companyId)
    → Returns: Promise<CompanyConfig>
    → Get company configuration from Firestore
  
  CompanyAdmin.applyBranding(config)
    → Apply colors/logo/name to DOM
    → Updates data-* attributes
  
  CompanyAdmin.getCurrentCompany()
    → Returns: Promise<CompanyConfig>
    → Get logged-in user's company
  
  CompanyAdmin.getCompanyId()
    → Returns: string ('nbd', 'oaks', etc.)
    → Get current company ID
  
  CompanyAdmin.setCurrentCompany(companyId)
    → Switch company context (admin use)

NBDAuth Extensions (pro/js/nbd-auth-enhancement.js)
  
  NBDAuth.initializeCompanyId()
    → Returns: Promise<string>
    → Load company ID from user document
  
  NBDAuth.getCompanyId()
    → Returns: string
    → Get current company ID
  
  NBDAuth.setCompanyId(companyId)
    → Set company ID (testing/admin)
